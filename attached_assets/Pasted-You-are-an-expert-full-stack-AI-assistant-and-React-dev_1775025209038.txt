Your goal is to generate a **complete USDT reward web app** scaffold that is **fully local, free-tier, and testable**, without any paid APIs or services. The app must include **React frontend + Tailwind CSS + Node.js/Express backend + PostgreSQL database**, covering **UX, UI, pool logic, wallet, transactions, admin panel, crypto deposit verification, celebration effects, tier/batch system, and profit-neutral gamification**.

Important Constraints:
- No paid APIs or paid services anywhere; only free tools allowed.
- Everything must be fully local-testable, including database, wallet logic, email verification (mock SMTP), pool participation, reward distribution, and tier logic.
- UX/UI must be **trustable, professional, and intuitive**, especially for crypto/finance users.
- Theme should be **slightly dark with modern gradients** (dark grey + green/blue highlights).

1. UX & Landing Page Improvements
- Build **trust with users**: clear CTAs, readable fonts, clear instructions.
- Landing page includes **pool stats, recent winners, upcoming pools**.
- Subtle **animations & hover effects** for buttons, pool cards, and winners.
- Include **popup notifications** for ticket purchase, pool win, and tier rewards.

2. Business Model & Ticket Commission
- Pool entry: **10 USD per ticket**.
- Pool capacity configurable by admin.
- App calculates **total commission** automatically.
- Transaction and wallet logic **reflect user deposits, pool contributions, and rewards**.

3. Pool Algorithm & Transparency
- Pools open/close transparently.
- Randomly select **3 winners** per pool.
- Each winner receives predefined rewards (e.g., 1st → 100 USDT, 2nd → 50 USDT, 3rd → 30 USDT).
- Admin can **override rewards** if needed.
- Pool and winner selection logic must be **auditable**.

4. Ticket Purchase Celebration
- When a user buys a ticket:
  - Show **popup or confetti animation** celebrating the purchase.
  - Show updated **wallet balance immediately**.
  - Indicate participation in pool with **countdown timer**.

5. Payment Handling & Charge
- Users can deposit via manual crypto deposit:
  - Add crypto exchange address + upload screenshot → Pending → Admin approves → wallet updated.
- Withdrawal handled similarly: user requests → pending → admin approves → wallet deducted.
- Popup notification for **successful deposit, withdrawal, or reward collection**.

6. Admin Panel & Full Control
- Pool management: create, edit, start, end, delete.
- User management: view profiles, wallet balances, deposits, withdrawals, crypto addresses, transaction screenshots.
- Transaction verification: approve/reject deposits and withdrawals.
- Reward distribution: automatic/random or manual override.
- Audit logs: all admin actions tracked.
- Dashboard with filters/search for users, pools, and transactions.

7. Backend Requirements
- Node.js + Express REST API.
- Routes for:
  - /auth: signup, login, logout
  - /users: get/update user info, crypto address, wallet balance, batch/tier info
  - /pools: get pools, join pool, distribute rewards
  - /transactions: deposits, withdrawals, rewards
  - /deposits: add crypto deposit, upload screenshot, approve/reject
  - /admin: pool management, user management, transaction approvals, audit logs
- Database: PostgreSQL (SQLite allowed for local dev)
- Wallet logic: manual but **auto-updated after admin approval**
- Security: validation, bcrypt password hashing

8. Tier / Batch System
- Users assigned to tiers based on activity points (deposits + tickets + engagement points).
- Suggested tiers: Aurora, Lumen, Nova, Celestia, Orion.
- Admin can **override tier manually**.
- **Gamified incentives** only: badges, profile highlights, progress bar, dashboard animations.
- **Profit-neutral**: no extra USDT reward required for tiers unless optional free ticket granted.
- Free ticket can be given **once per tier upgrade** to motivate users.
- Backend logic updates batch automatically, stores tier, and prevents repeat free ticket for the same tier.

9. UX / Frontend for Tiers
- Show tier badge and progress in user dashboard: “You are Aurora tier, 40% progress to Lumen.”
- Celebration animation when reaching a new tier.
- Highlight user tier in pools and recent winners feed.
- Optional leaderboard by tier points (cosmetic only, no financial effect).
