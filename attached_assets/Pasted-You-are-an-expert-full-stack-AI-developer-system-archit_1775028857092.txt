You are an expert full-stack AI developer, system architect, and UX/UI designer. Your goal is to design and implement a **production-grade USDT Reward Web App** that is **secure, scalable, trustable, and fully testable locally**, based on an incentivized reward pool model (NOT gambling).

Your implementation must handle **real-world edge cases, financial safety, user trust, and system reliability**.

--------------------------------------------------

🔷 CORE BUSINESS MODEL
- Users join pools by depositing USDT.
- Each pool has limited slots and a time limit.
- A random winner is selected.
- Winner gets the majority of funds.
- Non-winners receive a **partial refund**.
- Platform earns via **transparent fees only**.
- Users should NEVER feel their money is lost unfairly.

- UX Requirements:
  - Dark theme with green/blue trust colors
  - Mobile responsive (wallet always visible)

  - Accessible (keyboard + screen reader support)

--------------------------------------------------

🔷 PHASE 2: POOL SYSTEM (ROBUST LOGIC)

- Pool Features:
  - Fixed slots + countdown timer
  - Multiple pools running simultaneously
  - Users can join multiple pools

- Critical Scenarios:
  - Prevent **overbooking (atomic slot locking)**
  - Handle **simultaneous joins (race conditions)**
  - Temporary slot reservation during payment

- Incomplete Pool Handling:
  - If not filled:
    - Auto-extend (max 2 times)
    - OR refund all users
    - If ≥80% filled → optional partial draw

- UX:
  - Show statuses: Filling / Full / Extended / Refunded
  - Live participant updates

--------------------------------------------------

🔷 PHASE 3: WALLET & MONEY SAFETY

- Wallet must support:
  - Instant updates

  - Deposit / Withdraw flows

- Safety Requirements:
  - Prevent **double credit (idempotent transactions)**
  - Prevent negative balance
  - Handle pending states (deposit/withdraw)

- Always:
  - Use backend as source of truth
  - Log every transaction
