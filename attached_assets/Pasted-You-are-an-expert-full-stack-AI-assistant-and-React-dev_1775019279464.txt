2. User Features
- Authentication: Sign Up / Login / Logout with email verification (mock SMTP allowed), password hashing (bcrypt)
- Profile Page: Name, email, wallet balance, total deposits, total rewards
- Dashboard: Active pools, countdown timers, wallet balance
- Pool Participation: Entry tickets (10 USDT per ticket)
- Recent Winners Feed: Build trust and transparency
- Transaction History: Deposits, withdrawals, rewards, status of each transaction
- Optional: Referral system

UX Rules:
- Wallet balance must always be visible
- Countdown timers for pool ends
- Clear call-to-action (CTA) buttons: Join Pool, Check Wallet, Withdraw, Deposit
- Winner feed should be visible to all users
- Transaction status (Pending, Completed, Failed) visible in user wallet history

3. Admin Panel Features
- Pool Management: CRUD operations, start/end pools
- User Management: View users, wallet balances, transactions
- Reward Distribution: Trigger randomized rewards (3 winners: 100, 50, 30 USDT)
- Transaction Management: Approve or reject deposits and withdrawals manually
- Dashboard: Pool stats, participant lists, total rewards distributed
- Fully testable locally

4. Backend Requirements
- Stack: Node.js + Express REST API
- Routes:
  - /auth: signup, login, logout
  - /users: get/update user info
  - /pools: get pools, join pool, distribute rewards
  - /transactions: deposits, withdrawals, rewards
  - /admin: pool management, transaction approval, reward triggers
- Database: PostgreSQL (SQLite allowed for local dev)
- Security: Data validation, bcrypt password hashing
- Wallet: Manual wallet operations (no paid APIs)
  - Deposit: User clicks deposit → backend marks as “Pending” → admin approves → wallet balance updated
  - Withdrawal: User requests withdrawal → admin approves → balance reduced → transaction marked “Completed”
  - Reward: Automatically credited on pool win
- Optional Enhancements: Logging for all wallet and pool actions, Testnet crypto integration in future

5. Database Schema (PostgreSQL / SQLite for local dev)
- Users: user_id, name, email, password_hash, wallet_balance, total_deposits, total_rewards, joined_at
- Transactions: tx_id, user_id, tx_type (deposit, withdraw, reward), amount, status (pending, completed, failed), created_at
- Pools: pool_id, entry_fee, max_users, start_time, end_time, status (open, closed, completed)
- PoolParticipants: id, pool_id, user_id, ticket_count, joined_at
- Optional: Referrals: id, referrer_user_id, referred_user_id, reward_amount
