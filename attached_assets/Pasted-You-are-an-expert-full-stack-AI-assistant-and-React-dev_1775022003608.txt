- No paid APIs or paid services anywhere; only free tools allowed.

New Requirements:

1. Transaction Screenshot Verification
- Users must upload a screenshot of their deposit or transaction.

- Admin can approve or reject the transaction.

- Upon rejection, the balance remains unchanged and the user sees a clear rejection message.

2. Full Admin Control Over Users
- Admin can view all users’ detailed profiles, including:
  - Recent deposits and withdrawals
  - Wallet balances
  - Crypto exchange addresses
  - Uploaded transaction screenshots
- Admin can approve/reject pending deposits and withdrawals directly from the user profile.
- Admin can manually adjust wallet balances if needed.
- All actions should be logged for auditing.

3. Admin Dashboard Enhancements
- Dashboard shows all pending transactions with screenshot previews.
- Admin can filter/search users by name, email, status, deposit/withdrawal amount.
- Admin can trigger reward distribution for pools or adjust payouts manually.
- Admin can view audit logs of all user and admin actions.
- All approvals automatically update user balances in real-time.

4. UX & Theme Guidelines
- Professional, clean, and finance/crypto style (white + green/blue)
- Countdown timers for pools
- Recent winners feed visible
- Clear CTA buttons for deposits, withdrawals, approvals, pool participation
- Subtle animations for pool completion, winner announcements, and approval actions
- Mobile responsive and easy for non-technical users to navigate

5. Backend Requirements
- Routes for deposits with screenshot upload:
  - /deposits/create → user submits screenshot
  - /deposits/list → list user deposits with status
  - /admin/deposits/approve → approve/reject deposit, auto-update wallet
- Routes for withdrawals with optional verification:
  - /transactions/withdraw → request, pending approval

- User management:
