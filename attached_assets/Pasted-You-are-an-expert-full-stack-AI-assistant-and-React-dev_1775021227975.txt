2. User Features
- Authentication: Sign Up / Login / Logout with email verification (mock SMTP allowed), password hashing (bcrypt)
- Profile Page: Name, email, wallet balance, total deposits, total rewards, crypto exchange address
- Dashboard: Active pools, countdown timers, wallet balance
- Pool Participation: Entry tickets (10 USDT per ticket)
- Recent Winners Feed: Build trust and transparency
- Transaction History: Deposits, withdrawals, rewards, status of each transaction
- Crypto Deposit Request:
  - User can add crypto exchange address
  - Upload screenshot of payment
  - Request marked as Pending until admin approves → wallet balance updated
- Withdrawals: User enters withdrawal amount + wallet address → pending approval → wallet updated after admin approval
- Optional: Referral system

3. Admin Panel Features (Full Control)
- **Pool Management:**
  - Create, edit, delete pools
  - Start/end pools manually
  - Set max participants, entry fee, and reward structure
  - Close pools early if needed
- **User Management:**
  - View all users, edit user info, reset passwords
  - Adjust wallet balances manually
  - Suspend or ban users if needed
  - Track referrals, rewards, deposits, and withdrawals
- **Transaction & Deposit Management:**
  - Approve/reject deposits and withdrawals
  - View user crypto addresses and uploaded screenshots
  - Manually credit/debit funds if needed
- **Reward Distribution:**
  - Trigger pool rewards manually or automatically
  - Adjust winner amounts or override for special cases
  - Track history of all rewards
- **Dashboard:**
  - View pool stats, participant lists, user activity logs
  - Recent deposits and withdrawals
  - Real-time updates for pending approvals
- **Advanced Controls:**
  - Export user, pool, transaction, and deposit data as CSV
  - Search and filter users/pools/transactions by date, status, or amount
  - Audit logs for all admin actions
  - Configurable notifications for deposits, withdrawals, and pool completion

4. Backend Requirements
- Stack: Node.js + Express REST API
- Routes:
  - /auth: signup, login, logout
  - /users: get/update user info, add crypto exchange address, admin controls
  - /pools: get pools, join pool, distribute rewards, admin CRUD
  - /transactions: deposits, withdrawals, rewards
  - /deposits: add crypto deposit, upload screenshot, approve/reject deposits
  - /admin: pool management, user management, transaction & deposit approvals, reward overrides, audit logs
- Database: PostgreSQL (SQLite allowed for local dev)
- Security: Data validation, bcrypt password hashing
- Wallet Logic: Manual, simple, easy for any user
  - Deposit: User submits → Pending → admin approves → wallet updated
  - Withdrawal: User requests → Pending → admin approves → wallet reduced
  - Reward: Automatically credited on pool win
- Optional Enhancements: Logging for all wallet, deposit, and pool actions, testnet crypto integration for future

5. Database Schema (PostgreSQL / SQLite)
- Users: user_id, name, email, password_hash, wallet_balance, total_deposits, total_rewards, crypto_address, status (active/suspended), joined_at
- Transactions: tx_id, user_id, tx_type (deposit, withdraw, reward), amount, status (pending, completed, failed), created_at
- CryptoDeposits: deposit_id, user_id, crypto_address, screenshot_url, amount, status (pending, approved, rejected), created_at
- Pools: pool_id, entry_fee, max_users, start_time, end_time, reward_structure, status (open, closed, completed)
- PoolParticipants: id, pool_id, user_id, ticket_count, joined_at
- AdminActions (Audit Logs): action_id, admin_id, target_type, target_id, action_type, description, timestamp
- Optional: Referrals: id, referrer_user_id, referred_user_id, reward_amount
